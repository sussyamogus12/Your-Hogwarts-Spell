# Анимированные звёзды + Библиотека заклинаний

## Что делаем

1. **Живые звёзды на фоне** — сделаем частицы более «магическими»: мерцание (пульсация яркости), плавный дрейф и разный размер/цвет (золото + искры). Добавим несколько крупных «звёзд-искр» с шлейфом-свечением.

2. **Общая библиотека заклинаний** — новый раздел, где вошедшие пользователи выкладывают свои заклинания (название + что оно делает), а все посетители видят их лентой.

3. **Вход в аккаунт** — публиковать может только вошедший пользователь (Email/пароль + вход через Google). Рядом с заклинанием показывается имя автора.

4. **Публикация прямо из результата ИИ** — после генерации заклинания появляется кнопка «Опубликовать в библиотеку» (заполняет название и действие автоматически).

- Наверху появится ссылка-навигация между «Главная» и «Библиотека», а также кнопка входа/аккаунта (в едином тёмно-золотом стиле факультетов).
- Карточки заклинаний в виде свитков с золотой рамкой, автор внизу.

## Аккаунт и публикация

- Страница `/auth` — вход и регистрация (Email/пароль + Google).
- При регистрации пользователь указывает **имя (ник)**, которое будет отображаться как автор.
- Форма «Добавить заклинание»: поля **Название** и **Что делает заклинание** (действие). Доступна только вошедшим.
- Кнопка «Опубликовать в библиотеку» в блоке результата ИИ открывает эту форму с уже заполненными полями.
- Каждый видит все опубликованные заклинания; удалять/редактировать можно только свои.

## Технические детали

- **Бэкенд**: включаю Lovable Cloud (база данных + аутентификация). База нужна для общей библиотеки — localStorage не подойдёт, т.к. заклинания должны быть видны всем.
- **Аутентификация**: Email/пароль + Google (через брокер Lovable). Таблица `profiles` (id → auth.users, display_name) с триггером авто-создания при регистрации.
- **Таблица `spells`**: `id`, `user_id`, `author_name`, `name`, `effect`, `created_at`. RLS: чтение — всем (`anon`+`authenticated`), вставка — только своему `user_id`, удаление/изменение — только автору. GRANT-ы на все нужные роли.
- **Чтение библиотеки**: публичный server function с publishable-ключом + узкая `TO anon` SELECT-политика; страница `/library` публичная (SSR), лента через TanStack Query.
- **Запись**: server function с `requireSupabaseAuth`; форма вызывает её через `useServerFn` + `useMutation`, затем инвалидирует список.
- **Маршруты**: `/library` (публичная лента + форма для вошедших), `/auth` (вход/регистрация), навигация в шапке, управляемый слой `_authenticated` не обязателен, т.к. библиотека публична, а форма гейтится по сессии инлайн.
- **Звёзды**: доработаю генерацию искр в `index.tsx` (и вынесу в общий компонент `StarfieldBackground`, чтобы фон был и на `/library`), добавлю keyframes мерцания/дрейфа в `styles.css`.
- **SEO**: заголовки/описания и `head()` для `/library`.

Существующая генерация заклинаний через твой ключ OpenAI остаётся без изменений.
