## Цель
Заставить ИИ и интерфейс выводить заклинания строго в формате:

```
[заклинание на латинице] ([заклинание на русском])
[описание, 40 слов]
```

Пример:
```
Wingardium Leviosa (Вингардиум Левиоса)
Заклинание, позволяющее объекту парить в воздухе.
```

## Что изменится

### 1. Промпты бэкенда (`src/routes/api/spell.ts`)
- Переписать `MATCH_SYSTEM_PROMPT` и `CREATE_SYSTEM_PROMPT`.
- Требовать ответ строго в две строки:
  1. `Latin Name (Русское название)`
  2. Описание ровно 40 слов.
- Для режима «Подобрать»: если заклинание из Гарри Поттера изначально на латыни, выводить латинское название и в скобках русское произношение/перевод.
- Для режима «Создать»: придумать латинское название и дать русский перевод/транскрипцию в скобках.

### 2. Парсинг результата (`src/routes/index.tsx`)
- Расширить тип `SpellResult`:
  ```ts
  type SpellResult = {
    latinName: string;
    russianName: string;
    description: string;
  };
  ```
- Переписать `parseSpell(raw)`:
  - Первая непустая строки — название.
  - Извлечь `latinName` и `russianName` по шаблону `Latin (Russian)`.
  - Если скобок нет — вся первая строка считается латинским названием, русское остаётся пустым.
  - Всё остальное — описание, обрезанное до 40 слов.
- Очистка от меток (`заклинание:`, `название:` и т.п.) сохраняется.

### 3. Отображение результата (`src/routes/index.tsx`)
- Заголовок карточки результата выводить как:
  ```
  Wingardium Leviosa (Вингардиум Левиоса)
  ```
- Описание — одна строка/абзац, максимум 40 слов.
- Кнопка «Опубликовать в библиотеку» должна передавать оба названия в поисковые параметры (`name=latin|russian` или два отдельных поля).

### 4. Библиотека (`src/routes/library.tsx`)
- При публикации из главной страницы разобрать поисковые параметры и подставить полное название в поле «Название заклинания» в формате `Latin (Russian)`.
- Поиск по библиотеке должен работать по этому составному названию и по описанию.
- Карточки в библиотеке уже показывают `spell.name` — достаточно, чтобы в `spell.name` хранилось полное `Latin (Russian)` название.

## Технические детали
- Без изменений в базе данных: таблица `spells` хранит `name` и `effect`, формат имени регулируется на уровне генерации и парсинга.
- Без новых зависимостей.
- Проверка: сгенерировать заклинание в обоих режимах и убедиться, что заголовок соответствует шаблону, а описание не превышает 40 слов.
