# План доработок

## 1. Отказоустойчивый ИИ (три движка подряд)

Файл: `src/routes/api/spell.ts`

Сейчас используется только DeepSeek. Сделаем цепочку с автоматическим переходом при ошибке:

1. **ChatGPT** (`OPENAI_API_KEY`, `https://api.openai.com/v1/chat/completions`, модель `gpt-4o-mini`)
2. **DeepSeek** (`DEEPSEEK_API_KEY`, `https://api.deepseek.com/chat/completions`, модель `deepseek-chat`)
3. **Встроенный ИИ Lovable** (`LOVABLE_API_KEY`, `https://ai.gateway.lovable.dev/v1/chat/completions`, модель `google/gemini-3-flash-preview`) — как последний резерв.

Логика: перебираем движки по очереди; если ключ отсутствует или запрос падает / возвращает не-OK / пустой ответ — пробуем следующий. Ошибку показываем пользователю только если не сработал ни один. Все три используют один и тот же OpenAI-совместимый формат запроса, поэтому вынесем вызов в общую вспомогательную функцию `callChatModel(...)`. Порядок и системные промпты остаются прежними.

## 2. Библиотека: избранное, рейтинг, поиск

### База данных (миграция)
Две новые таблицы в `public`:

- `spell_votes` — `id`, `spell_id` (→ spells, on delete cascade), `user_id`, `value` (smallint: 1 лайк / -1 дизлайк), `created_at`; уникальность `(spell_id, user_id)`.
- `spell_favorites` — `id`, `spell_id`, `user_id`, `created_at`; уникальность `(spell_id, user_id)`.

Для каждой: GRANT'ы, RLS. Голоса/избранное видны всем на чтение (для подсчёта), но менять может только владелец строки (`auth.uid() = user_id`).

### Сервер (`src/lib/spells.functions.ts`)
- `listSpells` расширяем: помимо полей заклинания возвращаем `likes`, `dislikes`, `score` (likes − dislikes). Подсчёт агрегируем из `spell_votes`. Публичный (anon) доступ сохраняется.
- Новая авторизованная `getMyReactions()` — возвращает для текущего пользователя карту его голосов и избранного (чтобы подсветить кнопки). Вызывается только когда пользователь вошёл.
- Новая `voteSpell({ spellId, value })` — ставит/меняет/убирает голос (повторный клик по той же кнопке снимает голос).
- Новая `toggleFavorite({ spellId })` — добавляет/убирает из избранного.

### Интерфейс (`src/routes/library.tsx`)
- **Поиск**: поле ввода сверху, фильтрует список по имени и описанию (без учёта регистра) на клиенте.
- **Сортировка**: по умолчанию сначала заклинания с лучшим рейтингом (по `score`, затем по дате).
- **Фильтр «Избранное»**: переключатель «Все / Избранное» (избранное доступно вошедшим).
- В карточке `SpellCard`: кнопки лайк/дизлайк со счётчиком и звезда «в избранное». Для гостей клик по ним предлагает войти. После голосования/избранного инвалидируем кэш `["spells"]` и реакции.

## 3. Эффект при подборе/создании заклинания

Файлы: `src/routes/index.tsx`, `src/styles.css`

Во время колдовства (`loading`) и при появлении результата фон плавно «оживает» — накладывается мягкое цветное свечение (тёплое золото/пурпур), не режущее глаза. Реализация: добавим управляемый оверлей-градиент поверх фона, который анимированно проявляется при `loading`/`result` через плавный переход прозрачности и мягкую пульсацию (новый keyframe `aura-pulse`). Используем существующие токены (`--gold`, `--emerald`, `--burgundy`) с низкой насыщенностью.

## 4. Параллакс звёзд

Файл: `src/components/StarfieldBackground.tsx`

Сейчас звёзды в `fixed`-контейнере. Добавим реакцию на прокрутку:
- Слушаем `window` scroll (через `requestAnimationFrame` для плавности).
- Большие звёзды смещаются медленнее прокрутки (коэффициент ~0.15 — «чуть медленнее»).
- Маленькие звёзды двигаются с половиной скорости (коэффициент ~0.5).
- Смещение применяем через `transform: translateY(scrollY * factor)` для двух групп, сохраняя существующие анимации мерцания.

---

### Технические детали
- Миграция строго: CREATE TABLE → GRANT (authenticated + service_role; anon SELECT для подсчёта рейтинга) → ENABLE RLS → POLICIES.
- Голосование/избранное — только авторизованные (`requireSupabaseAuth`), клиент уже прикрепляет токен через `functionMiddleware`.
- `getMyReactions` вызывается отдельным запросом только при наличии пользователя, чтобы публичный SSR-лоадер не требовал токена.
- Порядок движков ИИ и промпты не меняются, только добавляется резервирование.
